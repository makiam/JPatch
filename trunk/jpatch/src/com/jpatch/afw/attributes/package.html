Attributes are wrappers around a primitives or objects. Unlike the primitive wrappers in java.lang (Integer, Boolean, Float, etc.), Attributes are deliberately mutable. The purpose of Attributes is to provide event notification when the wrapped value was changed. Unlike the event processing in the AWT and in Swing, Attributes provide pre- and post-event-notifications.

The classes in this package are generally not thread-safe!

Besides the "primitive" Attributes BooleanAttr, IntAttr and DoubleAttr there are spicialized versions of DoubleAttr (HardBoundedDoubleAttr and SoftBoundedDoubleAttr), a GenericAttr (that wraps around any object), Attributes that represent Tuples of 2 or 3 DoubleAttrs, a CollectionAttr that wraps a Collection and finally a StateMachine.

Attributes are especially useful for binding user-interface components like TextFields or Sliders to attribute-values. A lot of useful classes for this purpos can be found in the com.jpatch.awf.ui package.
